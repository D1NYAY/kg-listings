# Чеклист проверки работоспособности KG Listings

## Перед запуском

- [ ] PostgreSQL запущен
- [ ] Создана БД `kg_listings`
- [ ] `.env` заполнен (DATABASE_URL, TELEGRAM_BOT_TOKEN, TELEGRAM_ADMIN_IDS)
- [ ] Выполнены `npm run db:migrate` и `npm run db:seed`

## Бот

- [ ] `/start` — приветствие и кнопки
- [ ] `/submit` — пошаговый сбор данных
- [ ] На каждом шаге валидация (название, фото, цена и т.д.)
- [ ] `/cancel` — отмена процесса
- [ ] `/mylistings` — список своих объявлений
- [ ] После подачи — объявление в статусе PENDING

## Модерация

- [ ] Админ получает сообщение с карточкой объявления
- [ ] Кнопки «Одобрить» и «Отклонить»
- [ ] Одобрение → статус APPROVED, пост в канал, уведомление пользователю
- [ ] Отклонение → статус REJECTED, уведомление пользователю

## Канал

- [ ] После одобрения пост появляется в канале
- [ ] В посте: фото, название, цена, описание, город, контакт
- [ ] Кнопка «Смотреть в каталоге» ведёт на Mini App

## Mini App

- [ ] Главная — лента карточек одобренных объявлений
- [ ] Поиск по названию
- [ ] Фильтр по категории и городу
- [ ] Страница товара — фото, описание, кнопка «Написать»
- [ ] Кнопка «Написать» открывает Telegram чат с продавцом
- [ ] Нижняя навигация: Главная, Категории, Подать, Избранное, Профиль
- [ ] «Подать» ведёт в бота
- [ ] Избранное — заглушка

## API

- [ ] `GET /api/listings` — только APPROVED
- [ ] `GET /api/listings/:id` — одно объявление
- [ ] `GET /api/categories` — список категорий
- [ ] `GET /api/cities` — список городов
- [ ] `GET /api/image/:listingId` — прокси фото из Telegram

## Примеры тестовых данных

Создай объявление через бота:
- Название: «iPhone 13»
- Фото: любое
- Описание: «Отличное состояние»
- Цена: 45000
- Категория: Электроника
- Город: Бишкек
- Контакт: @твой_username
- Имя: Тест

После одобрения оно должно появиться в канале и Mini App.
