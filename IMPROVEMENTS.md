# Возможные улучшения после MVP

## Хранение состояния бота

- **Сейчас**: in-memory Map, сбрасывается при перезапуске
- **Улучшение**: Redis или таблица `Draft` в БД
- Плюс: пользователь может продолжить подачу после рестарта бота

## Фото

- **Сейчас**: только `telegram_file_id`, прокси через API
- **Улучшение**: скачивать при одобрении и сохранять в `uploads/` или S3
- Плюс: не зависим от Telegram, быстрее загрузка в Mini App

## Избранное

- **Сейчас**: заглушка
- **Улучшение**: таблица `Favorite`, привязка к `userId` (Telegram)
- В Mini App — кнопка сердечка, страница избранного

## Профиль

- **Сейчас**: только отображение данных из Telegram
- **Улучшение**: настройки, редактирование контактов, история объявлений

## Безопасность

- Rate limiting на API (express-rate-limit)
- Валидация и санитизация (XSS, инъекции)
- Ограничение размера загрузок

## UX

- Пагинация в ленте (бесконечный скролл)
- Фильтр по цене (от–до)
- Сортировка (по дате, цене)

## Админ-панель

- **Сейчас**: модерация только в боте
- **Улучшение**: простая web-страница с логином
- Список pending, approve/reject, статистика

## Уведомления

- Напоминание пользователю, если объявление не модерировано 24 часа
- Push при одобрении/отклонении (уже есть в боте)

## Аналитика

- Счётчики просмотров
- Конверсия одобрений
